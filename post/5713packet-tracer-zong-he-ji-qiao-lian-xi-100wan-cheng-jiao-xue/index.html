5.7.1.3：Packet Tracer 综合技巧练习100%完成教学

2022-04-24

注意：
本教程只限于将pak完成至100%进度，如100%测试ping命令未能ping通许请先自行检查。使用版本8以上cisco完成时需等待一段时间进行响应方可完成连通性测试。版本6cisco反应速度很快。完整代码适用新老两版的综合技巧练习。该代码为up个人总结，仅供学习使用，未经允许禁止转载！
简介：
本次练习着重于子网划分、基本设备配置、静态路由和 RIP 路由。当所有设备配置完毕后，您需要测试端到端的连通性并检查配置。
学习目标：
根据要求设计一个编址方案并记录下来。
在设备上应用基本配置。
配置 ISP 路由器间的静态路由。
在区域 1 和区域 2 中配置 RIPv1 路由
在适当的接口上禁用 RIP 更新
配置默认路由，然后通过 RIP 重分布路由信息。
检验拓扑中所有设备间的完全连通性。
任务 1：设计并记录编址方案。

步骤 1 － 设计编址方案。
根据拓扑和下列要求设计一个编址方案：
R1 与 R2 之间以及它们各自 ISP 路由器之间的 WAN 链路已经配置好。同样，ISP 和 Web 服务器之间的链路也已配置好。
由于 RIPv1 是有类路由协议，因此不能使用可变长子网掩码 (VLSM)。根据以下信息对每个区域的地址空间划分子网：
区域 1 地址空间中的最大子网能容纳 1,000 台主机。对于此 10.1.0.0/16 地址空间，应该使用什么子网掩码？
区域 2 地址空间中的最大子网能容纳 500 台主机。对于此 172.20.0.0/16 地址空间，应该使用什么子网掩码？
对于区域 1 中的 LAN，将子网 0 分配给连接到 B1-R1 的 FastEthernet 0/0 接口的 LAN。依次为其它 LAN 分配子网。子网 1 分配给连接到 B1-R1 的 FastEthernet 0/1 接口的 LAN；子网 2 分配给 FastEthernet 1/0；子网 3 分配给 FastEthernet 1/1……以此类推。
对于区域 1 中的 WAN，将最后一个子网分配给 R1 和 B3-R1 之间的链路，第二个到最后一个子网分配给 R1 和 B2-R1 之间的链路，第三个到最后一个子网分配给 R1 和 B1-R1 之间的链路。
在打印说明上的表格中写下区域 1 的子网分配情况。
对于区域 2 中的 LAN，按照与区域 1 相同的方式来分配子网：子网 0 分配给 B1-R2 上的 Fa0/0 接口；子网 1 分配给 Fa0/1……以此类推。
对于区域 2 中的 WAN，将最后一个子网分配给 R2 和 B3-R2 之间的链路，第二个到最后一个子网分配给 R2 和 B2-R2 之间的链路，第三个到最后一个子网分配给 R2 和 B1-R2 之间的链路。

任务 3：应用基本配置。
使用您所记录的文档在路由器上配置地址等基本配置。使用 cisco 作为线路口令，并使用 class 作为加密口令。使用 64000 作为时钟频率。
//R1和R2都做此配置
int s0/0/0
clock rate 64000
int s0/0/1
clock rate 64000
int s0/1/0
clock rate 64000

任务 4：配置 ISP 路由器之间的静态路由。
每台 ISP 路由器已配置有两条静态路由，可通往对方 ISP 路由器的直接相连的 WAN。在每台 ISP 路由器上实施静态路由，以确保两个区域之间的连通性。
//ISP-R1
ip route 10.1.0.0 255.255.0.0 s0/0/0
ip route 172.20.0.0 255.255.0.0 s0/0/1
//ISP-R2
ip route 172.20.0.0 255.255.0.0 s0/0/0
ip route 10.1.0.0 255.255.0.0 s0/0/1

任务 5：在区域 1 和区域 2 中配置 RIPv1 路由。
在所有区域性路由器上配置 RIP 路由。记住，ISP 路由器只使用静态路由。
//R1
router rip
version 1
network 10.0.0.0
default-information originate
passive-interface s0/1/1
exit
ip route 0.0.0.0 0.0.0.0 s0/1/1
//R2
router rip
version 1
network 172.20.0.0
default-information originate
passive-interface s0/1/1
exit
ip route 0.0.0.0 0.0.0.0 s0/1/1

任务 6：在适当的接口上禁用 RIP 更新。
并不需要通过所有的路由器接口发送 RIP 更新。在适当的接口上禁用 RIP 更新。
//B1\2\3-R1三个路由器
router rip
version 1
network 10.0.0.0
passive-interface f0/0
passive-interface f0/1
passive-interface f1/0
passive-interface f1/1
//B1\2\3-R2三个路由器
router rip
version 1
network 172.20.0.0
passive-interface f0/0
passive-interface f0/1
passive-interface f1/0
passive-interface f1/1
//任务5中R1、R2路由器已禁用相应端口的RIP更新

任务 7：配置默认路由，然后通过 RIP 重分布路由信息。
确定哪些路由器需要默认路由。然后配置该路由器，以向区域内的其它路由器重分布默认路由。
//任务六中network已经配置了默认路由。

任务 8：检验拓扑中所有设备间的完全连通性。
步骤 1 － 测试连通性。
现在，网络应该实现了端到端连通。请使用 ping 命令测试整个网络的连通性。每台路由器都应该可以 ping 通所有其它路由器接口以及两台 Web 服务器。
排除出现的网络故障，直到可以成功 ping 通。
步骤 2 － 检查配置。
使用检验命令确保配置完整。
